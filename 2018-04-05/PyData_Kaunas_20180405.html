Analysis of Lithuanian texts:
a case of moon and femininity

evaldas.vaiciukynas@ktu.lt

5 April 2018

Abstract

Presentation will discuss machine learning task of text classification. Text corpora was ASTRA stenograms, containing 110905 Lithuanian parliamentary transcripts from 147 speakers, collected during 1990 March - 2013 December. Texts were categorized by the political partisanship of a speaker, the gender of a speaker and the fact that a transcript was recorded around a full moon date. Types of pre-processing considered: original text, lemmized, morphized and translated to English. Lemmas and morphemes were obtained using semantika.lt and English translation using Google Translate services. Feature sets investigated: 6 from gensim (3 Doc2Vec variants, LSI, LDA, RP), 1 from fastText (Sent2Vec), and 3 custom-made (morfologija, stilometNER, ontologija). Random forest was used as a base-learner as well as a meta-learner (in 7 “stacking” configurations). Experiments reveal which categories, which types of pre-processing and which feature sets appear to be the most successful for texts analysed.

Types of text pre-processing applied

Before removal of punctuation and splitting into separate words, a few transformations were considered:

  • original - initial text “as is” without any pre-processing
    • Mano supratimu, šis dokumentas, kurį jūs teikiate, atsirado dėl tam_tikrų politinio veikimo problemų ir kažin_ar čia Seimui reikėtų pulti ir tvarkytis.
  • lemmized - text with words converted to their canonical form
    • aš supratimas , šis dokumentas , kuris tu teikti , atsirasti dėl tam_tikrų politinis veikimas problema ir kažin_ar čia seimas reikėti pulti ir tvarkytis .
  • morphized - morphological part of speech tag attached to lemma
    • aš_Pg-sgn supratimas_Ncmsin- šis_Pgmsnn dokumentas_Ncmsnn- kuris_Pgmsan tu_Pg-pnn teikti_Vgmp2p--n--ni- atsirasti_Vgms3p--n--yi- dėl_Sgg tam_tikrų_X- politinis_Agpmsgn veikimas_Ncmsgn- problema_Ncfpgn- ir_Cg kažin_ar_X- čia_Cg seimas_Ncmsdn- reikėti_Vgm-3---n--ns- pulti_Vgi-----n--n-- ir_Cg tvarkytis_Vgi-----n--y--
  • english - text translated to English language using Google Translate
    • In my understanding, this document, which you are providing, has arisen due to some political problems and somehow the Seimas should be attacked and managed.

Note: semantika.lt was used for lemmized and morphized, googletrans 2.2.0 for english.

Universal feature sets (6 gensim + 1 fastText)

  • 3 Doc2Vec - distributed bag of words model without (DBoW) and with (DBoWs) skip-gram, distributed memory paragraph vector model (DMPV) (Le and Mikolov, 2014)
    • Doc2Vec(size=64, dm=0, dbow_words=0, negative=5, hs=0)
    • Doc2Vec(size=64, dm=0, dbow_words=1, window=5, negative=5, hs=0)
    • Doc2Vec(size=64, dm=1, window=5, negative=5, hs=0)
  • LSI - latent semantic indexing, latent semantic analysis (Deerwester et al., 1990)
    • LsiModel(*,num_topics=64, chunksize=100, onepass=False, power_iters=3, extra_samples=100)
  • LDA - latent Dirichlet allocation (Blei et al., 2003)
    • LdaMulticore(*, num_topics=64, chunksize=100, passes=3, workers=3, eval_every=None)
  • RP - random projection, random indexing (Kanerva et al., 2000)
    • RpModel(*, num_topics=64)
  • S2V - sentence to vector (Pagliardini et al., 2017)
    • ./fasttext sent2vec -input ASTRA.txt -output ASTRA -epoch 10 -dim 64

Custom-made LT-specific feature sets

Feature sets, specific for Lithuanian language, were constructed after texts were annotated using semantika.lt services Various features were devised, such as:

  • morfologija (5277) - combinations for part of speech (POS) statistics
    • participant, noun, adjective, verb, pronoun, numeral, adverb…
  • stilometNER (57) - stilometry (27) and named entity recognition (30)
    • sentence count, total word count, unique-to-total ratio, person, organization…
  • ontologija (111) - counts of detected concepts in a predefined ontology
    • organization, person, currency, price, ironical phrase…

The largest feature set with 5277 elements was compacted by removing non-representative features. Feature (column) was removed if most of its values were zero, i.e. when less than 20% of values (rows) were non-zero. After screening sparse matrix with this rule only 364 features remained in morfologija feature set. Total number of feature sets (4 modalities x 7 universal + 3 custom) was 31.

Base-learner and meta-learner: random forest

Random forest classifier was an ensemble of 1000 unpruned CART trees. Why random forest? No overfitting, bagging gives OOB votes (no need for CV), simple tuning, easy to parallelize and stack. Decision-level fusion (aka. “stacking”):

  • noFusion (64) - the best single base detector as a baseline
  • origOnly (7) - all base detectors from original modality
  • origLemm (14) - all base detectors from original and lemmized modalities
  • origMorph (14) - all base detectors from original and morphized modalities
  • origEng (14) - all base detectors from original and english modalities
  • bestOnly (4) - the best base detectors from each modality
  • bestExtra (7) - the best base detectors with LT-specific detectors
  • fullFusion (31) - all base detectors from all modalities with LT-specific detectors

Evaluation of detection performance

  • CXE - cross-entropy between predicted class probability and actual class fact
  • EER - equal error rate, detector’s operating point where miss rate becomes equal false alarm rate (or sensitivity becomes equal to specificity)
  • Kappa - accuracy, corrected with respect to the class imbalance and random guessing
  • ROC plot - receiver operating characteristic curve (sensitivity vs. false alarm)
  • DET plot - detection error trade-off curve (ROC with logarithmic axes)
  • PR plot - precision-recall curve (complimentary to ROC or DET)
  • F1 - F-measure or geometrical mean between Precision and Recall
  • AUC - area under the ROC (receiver operating characteristic) curve
  • confusion matrix - predicted vs. actual class cross tabulation
    • http://www.marcovanetti.com/pages/cfmatrix/

CXE, EER, and AUC were obtained using Albayzin toolbox (Matlab).

Kappa, F1, confusion matrix were obtained using caret package (R).

DET curves were obtained using ROC, PR curves using precrec (R).

Conclusions

  • decision-level fusion (stacking) is better then the best single detector
    • for Moon class EER improved from 44.79% to 24.09%
    • for Female class EER improved from 40.31% to 34.98%
    • for Center class EER improved from 38.56% to 35.46%
    • for Right class EER improved from 40.79% to 36.46%
    • for Left class EER improved from 43.38% to 37.33%
  • after stacking, Moon class was more easy to predict than other classes
  • the overall best feature set was S2V (Sent2Vec modification of fastText)
  • custom-made morfologija, surprisingly, outperformed LSI feature set
  • lemmized better than morphized, but not when stacking with original
  • stacking original with english is also good (no need for semantika.lt)
